Usage

Découvrez comment utiliser facilement SassSwing pour créer des sites web responsive.

Téléchargement

SassSwing se présente sous deux formes différentes.
Sélectionnez ci-dessous, la version que vous souhaitez en fonction de vos préférences.

CSS

Il s'agit de la version standard avec les fichiers CSS minifiés et non minifiés. Contient également une version de base, contenant juste les composants de base.

Pour les utiliser il vous suffira de lier /styles/sass-swing.min.css dans le <head> de votre page HTML.

<link rel="stylesheet" href="./styles/sass-swing.min.css">
SASS SWING

SASS

Vous avez deux possibilités. Elles contiennent les fichiers source SCSS. En choisissant une de ces deux version,vous pourrez choisir les composants à inclure et vous aurez un controle total sur la personnalisation.

Générez un modèle, à partir de notre "Starter Kit" pour commencer à utiliser SassSwing directement pour construire votre site web (nécessite un compte Github)
Téléchargez ou clonez les sources pour nous aider et contribuer au développement de SassSwing (contient les fichiers source de cette documentation). 

>> git clone https://github.com/sassswing/sassswing.git
GENERER MODELSOURCE

CDN

Vous pouvez utiliser SassSwing directement dans vos productions via le serveur CDN unpkg ou js-deliver

<link rel="stylesheet" href="https://unpkg.com/@sass-swing/dist/css">

NPM

Vous pouvez également installer les sources de SassSwing via NPM

>> npm install sass-swing

Mise en place CSS

Cette section est pertinente que si vous avez choisi de télécharger la version CSS de SassSwing

Structure du projet

Si vous avez téléchargé, les fichiers CSS, il vous faut extraire les fichiers dans le dossier ou est situé votre projet.

La structure de votre dossier devrait ressembler à ceci : 

						
	  Mon-site-web/
	  |--styles/
	  |  |--sass-swing.css
	  |
	  |--assets/
		|  |--images/
		|     |--mon-image.jpg
		|  |--icons/
	  |  |--fonts/
		|     |--roboto
	  |
	  |--scripts/
	  |  |--sass-swing.js
	  |
	  |--index.html

Il existe deux ensembles de fichiers CSS. Le .min.css signifie que le fichier est 'compressé' pour réduire les temps de chargement. Ces fichiers minifiés sont généralement utilisés en production alors qu'il est préférable d'utiliser les fichiers non minifiés lors du développement.

Configuration HTML

Il vous suffit de vous assurer de lier correctement les fichiers dans votre page Web. Il est recommandé d'importer les fichiers javascript à la fin afin de réduire le temps de chargement de la page.

Suivez l'exemple ci-dessous : 

<!doctype html>
	<html lang="fr">
	  <head>
		  <meta name="viewport" content="width=device-width, initial-scale=1.0 minimum-scale=1.0">
		  <meta http-equiv="X-UA-Compatible" content="ie=edge">
		  <meta charset="UTF-8">
		  <title>Sass Swing documentation</title>
		  <meta name="Description" content="Sass Swing documentation"/>

		  <meta name="robots" content="index, follow">

		  <link rel="stylesheet" href="./styles/sass-swing.min.css">
		  <link rel="shortcut icon" href="./assets/icons/favicon.svg" type="image/svg">
	  </head>
	  <body>
	    <main class="container">
	      <h1>Hello, world!</h1>
	    </main>
	  </body>
	</html>

Mise en place SASS

Cette section est pertinente que si vous avez choisi de télécharger une des versions SASS de SassSwing

Si vous avez choisi de de générer le model, vous devrez impérativement cloner votre dépôt Github en local sur votre ordinateur

>> git clone https://github.com/username/repository

Ensuite, que vous ayez choisir une des deux version SASS de SassSwing, vous devez installer les dépendances NPM pour pouvoir compiler les fichiers SCSS en fichiers CSS exploitables pas vos pages HTML avec les commandes adequats

>> npm install sass-swing

Le modèle de projet

Structure du projet

						
	  Mon-site-web/
	  |--dist/
	  |
	  |--src/
		|  |--assets/
	  |     |--images/
		|     |--icons/
	  |     |--fonts/
	  |
	  |--scripts/
	  |  |--vendors
		|     |--sass-swing
	  |  |--index.js
		|
	  |--styles
		|  |--index.scss
		|  |--vendors
		|     |--sass-swing
		|
	  |  |--index.html

Les fichiers sources de SassSwing que vous allez utiliser, se trouvent dans les dossiers vendors/sass-swing.
Le dossier ./dist contiendra vos fichiers compilés

Commandes disponibles

  • Compilation mode dev

    Les fichiers seront simplement compilés, ils ne seront pas optimisés et compressés

    >> npm run dev
  • Compilation mode prod

    Les fichiers seront compilés, optimisés et minifiés.
    Seules les classes utilisées dans vos pages HTML, seront ajoutés au(x) fichier(s) css, final. Puis enfin le(s) fichier(s) seront minifiés

    >> npm run build
  • Compilation automatique en mode dev

    A chaque fois qu'un fichier sera modifié, la compilation en mode dev se lancera automatiquement

    >> npm run watch
  • Live server

    Permet de lancer un serveur de développement.
    A chaque modification, la compilation en mode dev se lancera et la page ouverte dans votre navigateur se rafraichira automatiquement

    >> npm run serve

Les sources

Cette distribution, contient uniquement les fichiers sources de Sass Swing ainsi que les fichiers de cette documentation

Commandes disponibles

  • Compilation de SassSwing

    Les fichiers compilés et compressés se trouveront dans le dossier ./dist

    >> npm run build
  • Génération des fichiers de la documentation

    Les fichiers compilés et compressés se trouveront dans le dossier ./documentation/styles/css

    >> npm run build:doc
